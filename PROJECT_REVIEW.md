## خلاصه فنی
- پروژه یک وب‌اپ PHP/MySQL برای مدیریت برنامه‌ریزی کنکور است.
- بک‌اند: PHP 8، PDO، Session، CSRF، bcrypt، JSON API
- فرانت‌اند: HTML/CSS/JS خام، RTL، PWA، بدون فریم‌ورک
- دیتابیس اصلی: MySQL با `utf8mb4`
- ماژول‌ها: پنل مشاور، پنل دانش‌آموز، برنامه‌ریز هفتگی، تسک‌ها، پیام‌رسانی، اعلان‌ها، حال روزانه، دستاوردها، آزمون‌ساز و آزمون‌دهی

## ساختار کلی
- `config/`: تنظیمات مرکزی و دیتابیس
- `includes/`: هسته مشترک شامل auth، db، helpers، layouts، models، planner settings، viewهای مشترک
- `auth/`: ورود، خروج، ثبت‌نام، انتظار تایید
- `admin/`: داشبورد، دانش‌آموزان، برنامه‌ها، برنامه‌ریز، گزارش‌ها، آزمون‌ها، پیام‌ها، تنظیمات، دستاوردها
- `student/`: داشبورد، برنامه، پیشرفت، پیام‌ها، آزمون‌ها، کارنامه، دستاورد، پروفایل، PDF برنامه
- `api/`: endpointهای AJAX برای تسک‌ها، پیام‌ها، آزمون‌ها، پروفایل، اعلان، mood و achievement
- `assets/`: CSS، JS، تصاویر، آیکن‌ها، فونت
- `sql/`: schema نصب اولیه و upgradeها
- `uploads/`: فایل‌های نمونه آزمون/پیام

## احراز هویت و امنیت
نقاط مثبت:
- استفاده از PDO prepared statements در اغلب مسیرها
- bcrypt برای پسورد
- CSRF برای فرم‌ها و APIهای تغییردهنده
- session cookie با HttpOnly، SameSite=Lax و Secure در HTTPS
- کنترل نقش در صفحات admin/student
- محدودیت آپلود و بررسی نوع فایل تصویری در آزمون‌ها

موارد قابل توجه:
- `install.php` و حساب‌های نمونه باید در production حذف/تغییر داده شوند.
- `auth.zip` داخل ریشه پروژه وجود دارد و بهتر است اگر نیاز نیست حذف شود.
- `uploads/` حاوی نمونه فایل است؛ برای دیپلوی تمیز باید تصمیم‌گیری شود.
- PHP CLI در محیط فعلی نصب نبود، بنابراین lint واقعی با `php -l` قابل اجرا نشد.

## نقاطی که در توسعه بعدی باید حواسمان باشد
1. بعضی آمارها مثل `advisor_stats()` ظاهراً global هستند و برای چند مشاور ممکن است نیاز به scope بر اساس advisor داشته باشند.
2. بعضی توابع مثل `all_subjects()` و دستاوردها در مدل فعلی بیشتر global رفتار می‌کنند؛ اگر چند مشاور مستقل داریم باید تفکیک داده تقویت شود.
3. در `student_exams()` توضیح تابع از بازه زمانی/assignment می‌گوید، ولی کوئری فعلی عمدتاً فقط `published` را می‌گیرد؛ اگر نیاز به start/end یا assign_all/assignment اختصاصی باشد باید اصلاح شود.
4. `install.php` علاوه بر نصب، upgradeهای زیادی را هم انجام می‌دهد؛ خوب است بعداً migrationها منظم‌تر شوند.
5. استفاده از `innerHTML` در JS زیاد است؛ بخش زیادی با escape کنترل شده، ولی هنگام افزودن داده جدید باید مراقب XSS باشیم.
